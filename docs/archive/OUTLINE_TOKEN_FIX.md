## 🎯 问题诊断

### 从日志发现的好消息

**API成功返回了正确的JSON格式！**
```json
{
  "title": "星河彼岸中",
  "chapters": [
    {"num": 1, "title": "意外的穿越", "description": "林夜在实验室事故中穿越时空..."},
    {"num": 2, "title": "陌生星域", "description": "林夜初步了解奥瑞亚星域的概况..."},
    ...
    {"num": 12, "title": "凯恩的阴谋", "description": "林夜发现凯恩正在收集星核碎片..."}
  ]
}
```

### 🐛 真正的问题

**响应被截断！**
```
错误: Expecting ',' delimiter: line 43 column 64 (char 3191)
最后的JSON: {"num": 12, "title": "凯恩的阴谋", "description": "林夜发现凯恩正在收集星核碎片，试图重启系统。"},
{"nu  ← 被截断！
```

**数据统计**:
- API响应长度：3582字符
- 生成内容：前12章完整 + 第13章被截断
- 需要的token数：约4000-6000 tokens
- **当前max_tokens限制：4000** ← 不够！

---

## ✅ 修复方案

### 增加max_tokens限制

**修改位置**: `src/ui/features/auto_generation.py` 第121-125行

**修改前**:
```python
max_tokens=4000  # ❌ 只能生成约12章
```

**修改后**:
```python
max_tokens=8000  # ✅ 可以生成50章完整大纲
```

### Token估算

根据日志分析：
- 12章大纲 ≈ 3582字符 ≈ 3000-3500 tokens
- 50章大纲 ≈ 15000字符 ≈ 12000-15000 tokens
- **设置8000 tokens应该足够生成50章**

**安全系数**:
- 8000 tokens 可以生成约 25-30章
- 加上JSON结构开销，应该能覆盖50章

## 💡 如果仍不够

**极端情况**: 如果8000 tokens仍然不够

**解决方案**:
1. **分批生成**: 先生成前30章，再生成后20章
2. **增加max_tokens**: 修改为16000（某些模型支持）
3. **减少章节数**: 从50章改为30章

**配置位置**:
```python
# src/ui/features/auto_generation.py 第124行
max_tokens=16000  # 极端情况
```

## ⚠️ 不同模型的Token限制

| 模型 | 最大tokens | 推荐max_tokens |
|------|-----------|----------------|
| GLM-4.5-air | 128000 | 8000-16000 |
| GLM-4-Flash | 128000 | 8000-16000 |
| GLM-4 | 8192 | 4000-6000 |
| GPT-4o-mini | 128000 | 8000-16000 |

**说明**:
- 不同模型的限制不同
- 智谱AI的GLM-4.5-air支持128K，所以8000很安全
- 如果使用更小的模型，可能需要调整
