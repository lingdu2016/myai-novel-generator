# API重试机制 - 快速使用指南

## 概述

现在所有API调用都支持自动重试，并且可以在接口管理中配置重试次数。

## 两个重要修复

### 1. ✅ 修复JSON解析失败

**问题**：AI返回的JSON包含中文引号导致解析失败
**解决**：自动将中文引号转换为英文引号
**效果**：大纲生成不再出现JSON解析错误

### 2. ✅ 添加自动重试机制

**功能**：API调用失败时自动重试
**配置**：在API配置中设置重试次数（0-10次）
**策略**：指数退避（2秒、4秒、8秒...）

## 如何配置

### 步骤

1. 打开应用
2. 进入 `系统设置` > `API配置`
3. 选择提供商（如OpenAI、Ollama等）
4. 展开 `高级设置`
5. 调整 `最大重试次数` 滑块
6. 点击 `💾 保存配置`

### 推荐配置

| 网络环境 | 重试次数 | 说明 |
|----------|----------|------|
| 稳定 | 1-2次 | API很少失败，快速失败即可 |
| 一般 | 3次 | 默认配置，适合大多数情况 |
| 不稳定 | 5次 | 网络经常波动 |
| 极差 | 8-10次 | 频繁断网 |

## 重试机制工作原理

### 重试流程

```
第1次调用 → 失败 → 等待2秒 → 第1次重试
         ↓ 失败 → 等待4秒 → 第2次重试
         ↓ 失败 → 等待8秒 → 第3次重试
         ↓ 成功 → 返回结果
```

### 等待时间（指数退避）

| 重试次数 | 等待时间 | 累计等待 |
|----------|----------|----------|
| 第1次 | 2秒 | 2秒 |
| 第2次 | 4秒 | 6秒 |
| 第3次 | 8秒 | 14秒 |
| 第4次 | 16秒 | 30秒 |
| 第5次 | 32秒 | 62秒 |

## 支持的功能

### ✅ 自动重试的场景

- 网络连接超时
- API服务器临时不可用

- API Key错误
- 权限不足
- 参数错误
- 模型不存在
- 这些错误重试也没用

## 常见问题

### Q1: 设置为0次是什么意思？
**A**: 不重试，API调用失败时立即返回错误。适合调试或网络非常稳定的情况。

### Q2: 设置为10次会不会太久？
**A**: 最多可能等待约20分钟（2+4+8+16+32+64+128+256+512秒）。建议不要设置太高，除非网络确实很差。

### Q3: 为什么有时候没有重试？
**A**: 某些错误（如API Key错误）重试也没用，所以不会重试。只有临时性错误才会重试。

### Q4: 可以在生成过程中修改配置吗？
**A**: 不行。配置修改后需要重新开始生成。建议提前配置好。

### Q5: 多个提供商怎么重试？
**A**: 每个提供商独立配置重试次数。系统会轮询所有提供商，每个失败后会尝试下一个。

## 技术细节

### 修复的JSON问题

**修复前**：
```python
# AI返回的JSON
{"description": ""大崩塌"实为宇宙重置程序"}  # 中文引号
# 解析失败 ❌
```

**修复后**：
```python
# 自动转换中文引号
cleaned_response = response.replace('"', '"').replace('"', '"')
# {"description": "\"大崩塌\"实为宇宙重置程序"}  # 英文引号
# 解析成功 ✅
```

### 重试算法

```python
retry_count = 0
while retry_count < max_retries:
    try:
        # 调用API
        return result
    except Exception as e:
        retry_count += 1
        if retry_count < max_retries:
            wait_time = 2 ** retry_count  # 指数退避
            time.sleep(wait_time)
raise Exception(f"已重试{max_retries}次，仍然失败")
```
